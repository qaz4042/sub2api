# 部署配置参考

这里放不属于日常发布流程、但部署时可能会查到的配置线索。入口命令仍以 `README.md` 和 `部署指南.md` 为准。

## 数据库迁移

- 启动时会按文件名顺序执行 `backend/migrations/*.sql`。
- 已执行迁移记录在 `schema_migrations` 表中，包含文件名和 checksum。
- 迁移默认只向前；回退需要恢复数据库备份，或编写补偿 SQL。
- `users.allowed_groups` 已迁移到 `user_allowed_groups` 联接表，相关校验可参考 `backend/migrations/README.md`。

## Gemini OAuth

部署侧主要关注环境变量：

```bash
GEMINI_OAUTH_CLIENT_ID=
GEMINI_OAUTH_CLIENT_SECRET=
GEMINI_OAUTH_SCOPES=
```

配置说明见 `.env.example` 和 `config.example.yaml` 的 Gemini OAuth 段落。AI Studio OAuth 需要自建 OAuth Client；Code Assist 模式可留空使用内置客户端。

## TLS Fingerprint

配置入口在 `config.example.yaml`：

```yaml
gateway:
  tls_fingerprint:
    enabled: true
```

账号侧可通过 `enable_tls_fingerprint` 和 `tls_fingerprint_profile_id` 选择是否启用指定指纹配置。

## systemd 二进制安装

通用安装脚本仍保留：

```bash
sudo ./install.sh
sudo ./install.sh upgrade
sudo ./install.sh uninstall
```

当前生产发布使用仓库根目录 Makefile：

```bash
make deploy-my4g
make deploy-my4g-backend-only
```
