# 生产问题本地调试

生产环境是 `ssh my4g`。本地排查默认不要同步完整生产数据和配置，只导出最小、只读、脱敏样本。

## 推荐流程

1. 先从日志、trace id、用户反馈和请求时间圈定问题范围。
2. 使用脚本导出脱敏小样本：

```bash
make export-my4g-debug-snapshot
```

默认是 `DRY_RUN=1`，只展示将要做什么。确认后执行：

```bash
DRY_RUN=0 \
CONFIRM_MY4G_DEBUG_EXPORT=I_UNDERSTAND_READONLY_REDACTED \
make export-my4g-debug-snapshot
```

输出目录默认在：

```text
.local-data/debug-snapshots/
```

## 脚本边界

- 不下载 `/opt/sub2api/sub2api.env`。
- 不执行 `pg_dump` 全库。
- 不导出明文密码、Token、Secret、API Key、OAuth credentials。
- 不写入生产数据库。
- 只导出 `settings` 摘要、表行数估算，以及 `users`、`api_keys`、`accounts`、`groups`、`usage_logs` 的小样本。

可调参数：

```bash
LIMIT_USERS=50
LIMIT_API_KEYS=100
LIMIT_ACCOUNTS=100
LIMIT_USAGE=300
USAGE_DAYS=7
SSH_TARGET=my4g
REMOTE_DIR=/opt/sub2api
```

## 仍然不能复现时

继续按“最小可复现上下文”补数据：只补相关用户、相关 API Key、相关账号、相关时间窗口的记录。不要把生产 `.env`、完整数据库、Redis dump 或对象存储内容拉到本地。
