# 费用公告

更新日期：2026-06-23
适用范围：本站公开售卖的 OpenAI 官方 API 直连接入服务

## 一、公告目的

本公告用于统一说明本站的费用口径、计费方式、价格基准、充值规则和异常处理原则，减少因市场口径不一致带来的误解。

如无特别说明，本站默认采用以下原则：

- 只做 OpenAI 官方 API 直连接入。
- 只按公开模型价格和站内余额规则计费。
- 不把 ChatGPT / Codex 订阅费用换算为 API 成本口径。

## 二、服务范围

本站当前提供的是：

- OpenAI 官方 API 直连接入能力
- 站内余额充值与扣费
- API Key 管理与调用
- 使用量与费用记录查询

本站当前不承诺：

- 所有 OpenAI 模型永久开放
- 所有模型始终同价
- 上游策略变化时完全不受影响

## 三、计费原则

### 1. 计费基准

模型费用以 OpenAI 官方 API 公布价格为基准维护。  
如官方价格调整、模型下线或计费口径变化，本站会同步更新。

参考来源：

- [OpenAI API Pricing](https://openai.com/api/pricing/)
- [OpenAI Developer Pricing](https://developers.openai.com/api/docs/pricing)

### 2. 计费单位

如无特别说明，模型消耗按 `USD / 1M tokens` 作为内部核算基准，站内按余额扣费。

实际展示口径可能包含：

- 模型输入费用
- 模型缓存输入费用
- 模型输出费用
- 订单支付金额
- 站内余额变动

上述项目分别代表不同维度，不应混为同一个价格概念。

### 3. 扣费时点

- 请求成功进入有效处理并产生可计量消耗后，按实际用量扣费。
- 仅支付成功不代表已发生模型扣费。
- 不同请求的最终扣费以站内使用记录和账单记录为准。

## 五、充值与余额规则

充值页面展示的是支付价格，模型扣费使用的是站内余额。  
这两者不是同一个概念，请以页面说明为准理解。

建议在正式运营时长期固定以下信息：

- 支付货币
- 余额单位
- 充值到账倍率
- 是否存在服务费或销售倍率

若采用示例口径，可写为：

```text
1 USD 站内余额 = 0.8 CNY 支付价格
支付 1 CNY = 到账 1.25 USD 站内余额
模型扣费按官方 API 价格基准执行
```

如果未来调整充值倍率、支付价格或售卖策略，应先更新本公告，再执行站内改价。

## 六、价格调整规则

出现以下情况时，本站可能调整价格或开放模型范围：

- OpenAI 官方价格变动
- 官方模型上下线
- 上游接口能力变化
- 站内运营成本明显变化
- 风险控制和售后成本明显变化

调整原则建议保持简单：

- 调整前先公告
- 公告中写明生效时间
- 尽量避免短期内频繁改价
- 已充值余额的使用规则保持连续可理解

## 七、异常与质保说明

本站主打“自用质保”，但质保必须以公开规则为准。

建议默认采用以下处理边界：

- 因站内计费异常造成的多扣费，经核查后补回对应余额。
- 因上游失败且未形成有效消耗的请求，不应重复扣费。
- 因官方接口波动、限流、模型不可用导致的中断，不承诺无限赔付，但会按已公开规则处理。
- 因用户自身代码错误、超量调用、密钥泄露造成的损失，通常不属于站内赔付范围。

如要进一步增强信任，建议后续单独补一份：

- `售后与异常处理规则`

## 八、重要说明

为避免理解偏差，请特别注意：

- ChatGPT / Codex 订阅价格不等于 OpenAI API Token 成本。
- 页面上的充值价格不等于模型单次调用价格。
- 余额是站内结算单位，不代表官方账户余额。
- 模型、价格、可用性都可能因官方策略变化而调整。

如你对费用口径、模型定价、异常扣费或余额规则有疑问，应优先通过站内公开客服入口咨询。
