# 客服标准回复

更新日期：2026-06-27

适用位置：站内客服、用户群、私聊咨询、工单回复。

## 一、价格咨询

回复：

> 费用请以站内费用公告为准。充值金额、站内余额和模型扣费是不同概念：充值是支付口径，余额是账户结算口径，模型扣费按实际 Token 消耗计算。建议先用轻量模型做一次最小测试，再接入正式项目。

## 二、是否官方

回复：

> 本站不是 OpenAI 官方网站，也不代表 OpenAI。本站提供的是基于 OpenAI 官方 API 的接入、计费、Key 管理和用量记录服务。模型可用性和价格会受到官方 API 变化影响。

## 三、怎么接入

回复：

> 第一次使用按四步走：注册登录、充值或兑换、创建 API Key、复制 Base URL 发起测试请求。建议先用使用指南里的 curl 示例跑通，再接入自己的项目。

## 四、Codex App 怎么用

回复：

> 可以参考开发者解决方案文档。一般流程是：在本站创建 API Key，点击导入到 CC Switch，切换配置后重启 Codex App，再用一个简单任务测试链路。

## 五、充值未到账

回复：

> 请提供账户标识、支付时间、支付金额、订单号或支付截图。我们会核查支付记录和站内订单状态。如果确认支付成功但余额未到账，会按规则补发。

## 六、请求失败是否扣费

回复：

> 是否扣费需要看请求是否已经形成有效上游消耗。未进入上游处理或未产生有效消耗的请求，不应按成功请求扣费。请提供请求时间、模型、错误信息和 request_id，我们会按记录核查。

## 七、感觉扣费高

回复：

> 高费用通常和模型单价、输入上下文长度、输出长度、历史消息数量有关。请先查看使用记录里的模型和 Token 消耗，再对照费用公告。如果仍有疑问，可以提供 request_id 核查。

## 八、API Key 泄露

回复：

> 请立即删除或禁用泄露的 API Key，并创建新 Key。同时检查近期使用记录，移除代码仓库、日志、截图和配置文件中的旧 Key。因用户自身泄露造成的消耗，通常不属于赔付范围。

## 九、模型不可用

回复：

> 实际可用模型以站内模型列表和公告为准。模型开放范围可能受官方模型上下线、价格变化、稳定性和运营成本影响。建议先使用当前页面可见的模型测试。

## 十、企业或私有部署咨询

回复：

> 支持私有部署到客户自有服务器或云账号中。标准交付通常包含服务部署、后台初始化、模型和 Key 配置、调用测试、备份说明和交接文档。具体费用需要看部署环境、定制范围和运维要求。

## 十一、不清楚问题在哪里

回复：

> 建议先用使用指南里的最小 curl 示例测试。如果 curl 成功，问题大概率在业务代码或工具配置；如果 curl 失败，请把错误信息、请求时间、模型和 request_id 发给我们排查。不要发送完整 API Key。
