# 文档目录

本文档按“developer”和“platform”两块整理。

## developer

面向开发、部署、运维、支付对接和技术风控。重点是系统如何搭建、运行、维护，以及当前代码和架构需要优先处理的问题。

- `operator/`：部署、运维、运行模式、支付配置、管理接口和平台配置管理。
- `internal/开源更新与能力扩展手册.md`：上游持续同步与私有能力扩展的目录、接入点、分支和检查清单。
- `internal/主干能力迁移代办.md`：从 `main` 重建定制分支并按旧分支历史迁移能力的执行清单。
- `internal/当前项目大问题与优先级建议.md`：项目风险、技术债和优先级建议。
- `internal/当前风控流程.md`：当前代码实现里的风控流程。
- `internal/标准风控机制.md`：建议落地的标准风控机制。
- `internal/风控中心配置与关键词库建议.md`：风控中心配置、关键词库和策略建议。

## platform

面向经营决策、运营准备、对外说明、客服话术、推广文案和合规承诺。重点是站点怎么卖、怎么运营、怎么向用户解释。

- `internal/`：商业化判断、站点运营思路、上线前配置与数据检查。
- `public/使用指南.md`：用户注册、充值、创建 API Key 和调用测试。
- `public/费用公告.md`：计费口径、充值规则和异常处理说明。
- `public/开发者解决方案.md`：开发者接入 Codex App / CC Switch 的说明。
- `public/企业解决方案.md`：对外展示的企业服务说明。
- `public/DISCLAIMER.md`：项目风险声明。
- `public/legal/`：可公开链接的合规承诺文本。
- `public/运营文案/`：官网、帮助中心、客服、推广和私有部署服务可复用文案。
