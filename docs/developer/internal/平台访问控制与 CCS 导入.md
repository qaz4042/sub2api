# 平台访问控制与 CCS 导入

## 平台访问控制

平台配置由迁移 `174_platform_configs.sql` 创建，默认支持 OpenAI、Anthropic、Gemini、Antigravity 和 Grok，且默认全部启用，以保持迁移前的行为。

管理员可在管理后台 **设置 → 平台访问控制** 中关闭非核心平台。OpenAI 是核心平台，不能关闭。关闭后，匹配该平台分组的网关请求返回 `403`；重新启用后，访问控制缓存立即失效，无需重启服务。

后台 API 也可用于自动化配置：

- `GET /api/v1/admin/platforms`：读取平台配置。
- `PUT /api/v1/admin/platforms/{key}`：使用 `{ "enabled": false }` 关闭平台，使用 `{ "enabled": true }` 重新启用。

上述接口需要管理员权限。未知平台或尝试关闭核心平台会返回错误，不会改变现有配置。

## CCS 导入地址

在管理后台 **设置 → 基本设置 → CCS 导入专用地址** 配置 `ccs_import_base_url`，适用于前端域名与 API 域名不同的部署。CCS 导入地址按以下顺序选择：

1. `ccs_import_base_url`；
2. `api_base_url`；
3. 当前站点来源。

系统会去除地址末尾的 `/`。留空时保持原有的 API 地址或当前站点回退行为。

## 回滚

优先在管理后台重新启用被关闭的平台。数据库迁移只追加表和默认数据，不需要删除表；若需要完全撤销此能力，应先恢复所有平台为启用状态，再由维护者按数据库变更流程处理，不直接在生产环境执行破坏性回滚。
