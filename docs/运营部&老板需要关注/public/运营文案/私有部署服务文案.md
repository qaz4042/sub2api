# 私有部署服务文案

更新日期：2026-06-27

适用位置：企业服务页、报价单、咨询回复、交付说明。

## 一、服务定位

推荐表达：

> 将 Sub2API 部署到客户自有服务器或云账号中，形成团队内部可管理、可统计、可交接的 AI API 网关。

## 二、适合对象

适合以下客户：

- 3-30 人开发团队。
- 需要统一 AI API 接入的工作室。
- 已有内部系统需要接入 OpenAI 风格 API 的企业。
- 不希望把核心 API Key 分散给每个成员的团队。
- 需要用量统计、额度控制和基础审计的团队。

## 三、标准交付内容

可作为标准私有部署包：

- Sub2API 后端部署。
- 前端管理台部署。
- PostgreSQL 和 Redis 配置。
- 域名和 HTTPS 配置。
- 管理员账号初始化。
- 基础模型、分组、额度和倍率配置。
- API Key 创建和测试。
- 备份策略配置。
- 基础运维文档和交接说明。

## 四、可选服务

根据实际需求可增加：

- 客户自有 API Key 接入。
- 企业内部账号系统对接。
- 业务系统接口联调。
- 日志脱敏和保留策略。
- 监控和告警配置。
- 数据迁移。
- 定制页面或品牌配置。
- 月度托管运维。

## 五、私有部署卖点

### 数据和权限更可控

部署在客户自己的服务器或云账号中，访问入口、数据库、日志和备份由客户控制。

### 团队用量更清楚

可以按项目或成员创建 API Key，减少多人共用一个 Key 带来的对账和排障问题。

### 交付结果可验收

交付不以抽象咨询为目标，而以可登录、可创建 Key、可调用、可查看用量、可备份恢复为验收基础。

## 六、报价沟通文案

建议对外使用固定边界报价：

```text
标准私有部署：包含服务部署、基础配置、管理员初始化、API 调用测试和交接文档。
不包含复杂业务系统改造、长期 7x24 运维、客户内部网络审批、上游账号申请和额外定制开发。
```

如果客户要求降价，建议同步删减范围，而不是在责任不变的情况下直接降价。

## 七、验收标准

标准私有部署建议以以下结果验收：

- 管理后台可以登录。
- 普通用户可以创建或使用 API Key。
- curl 示例可以成功调用。
- 使用记录可以查询。
- 余额或额度变化符合配置。
- 数据库和配置备份方式已说明。
- 交接文档已提供。

## 八、风险说明

私有部署客户应自行确认：

- 上游 API 或账号使用符合服务条款。
- 部署地、用户所在地和业务所在地合规要求。
- 数据安全、内容安全和日志留存要求。
- 是否需要备案、合同、发票、隐私政策或用户协议。

Sub2API 是软件和部署服务，不应被表述为对上游服务可用性、价格或政策不变的担保。
