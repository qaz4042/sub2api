# 使用指南

更新日期：2026-06-23
适用对象：首次使用本站 OpenAI 官方 API 接入服务的用户

## 一、先看这三件事

在开始前，建议先确认：

- 你是否已经阅读过[费用公告](./费用公告.md)。
- 你是否清楚本站提供的是 OpenAI 官方 API 直连接入服务。
- 你是否知道充值金额、站内余额和模型扣费不是同一个概念。

如果这三点已经清楚，再开始实际接入会更顺。

## 二、使用流程总览

第一次使用，按下面四步走即可：

1. 注册并登录账号
2. 充值或兑换余额
3. 创建 API Key
4. 复制 Base URL 并发起测试请求

建议先用一个最简单的文本请求确认链路跑通，再接入你自己的项目。

## 三、详细步骤

## 3.1 注册并登录

- 打开站点首页或登录页。
- 完成注册、登录或第三方登录。
- 首次登录后，先进入个人 Dashboard 查看账户状态。

如果站点处于邀请制或封闭测试阶段，请先确认自己是否已获得使用权限。

## 3.2 充值或兑换

- 进入充值页或兑换页。
- 按页面说明完成支付或输入兑换码。
- 到账后，先确认站内余额是否已更新。

注意：

- 支付成功不代表已经发生模型扣费。
- 充值金额和模型实际消耗是两套不同口径。
- 如对到账倍率或余额规则有疑问，请先查看费用公告。

## 3.3 创建 API Key

- 进入 `API Keys` 页面。
- 创建一个新的 Key。
- 妥善保存该 Key，不要泄露给他人。

建议：

- 不同项目使用不同 Key，便于排查和统计。
- 不要把 Key 直接提交到 Git 仓库。
- 如怀疑泄露，立即删除并重新生成。

## 3.4 获取接入信息

你通常需要两项信息：

- `Base URL`
- `API Key`

其中：

- `Base URL` 由本站提供
- `API Key` 为你在站内创建的密钥

如果站点兼容 OpenAI SDK，一般只需要把默认的 OpenAI Base URL 改成本站提供的地址，再填入你的 Key 即可。

## 四、快速测试

建议先做一次最小请求，确认是否接通。

示例：

```bash
curl https://your-domain.com/v1/responses \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "model": "gpt-5.4-mini",
    "input": "你好，请返回一行测试成功。"
  }'
```

如果返回正常内容，说明：

- 账户可用
- Key 可用
- 基础链路可用
- 模型映射基本正常

正式使用时，请把 `https://your-domain.com` 和 `YOUR_API_KEY` 替换成站内实际信息。

## 五、接入建议

为了减少初次使用的排错成本，建议这样接：

- 先用 `gpt-5.4-mini` 做首个测试。
- 先跑文本请求，不要一开始就接复杂多模态能力。
- 先在本地或测试环境跑通，再接到正式业务。

这样可以更快区分问题出在：

- 账户余额
- API Key
- Base URL
- 模型名
- 代码逻辑

## 六、如何查看用量和费用

调用后，你可以在站内查看：

- 使用记录
- 模型消耗
- 余额变化
- 订单或充值记录

如果你发现“返回成功”和“费用变化”理解不一致，优先对照：

- 请求是否真的产生了有效输出
- 使用记录中的模型与 Token 消耗
- 费用公告中的计费口径

## 七、常见问题

### 1. 为什么我已经充值了，还会提示余额不足？

可能原因包括：

- 充值尚未到账
- 充值到账金额与预期不一致
- 当前调用模型单价较高
- 账户已有其他消耗

应先检查余额页、订单页和费用公告。

### 2. 为什么我有 Key，但还是调用失败？

常见原因包括：

- Key 复制错误
- Base URL 填错
- 模型名写错
- 该模型当前未开放
- 请求格式不符合兼容要求

建议先用上面的 `curl` 示例做最小排查。

### 3. 为什么一次调用费用比想象中高？

常见原因包括：

- 输入上下文过长
- 输出内容过多
- 使用了更高单价模型
- 把充值价格和 Token 计费价格混在一起理解

这也是为什么建议先看费用公告。

### 4. 为什么站内模型和 OpenAI 官网展示不完全一致？

因为站点可能只开放部分模型，或出于稳定性和维护成本考虑，对开放范围做了筛选。  
是否可用，以站内当前可见模型和公告为准。

## 八、安全建议

- 不要把 API Key 发给他人。
- 不要把 Key 写进前端公开代码。
- 不要在公共仓库提交 `.env` 中的密钥。
- 定期轮换高频使用的 Key。

如果你怀疑 Key 已泄露，最安全的做法是立即删除旧 Key 并创建新 Key。

## 九、推荐阅读顺序

如果你是第一次接触本站，建议按这个顺序看：

1. [费用公告](./费用公告.md)
2. 本使用指南
3. 站内接口文档
4. 常见问题或客服入口

## 十、结语

本站的目标不是把接入流程做得很复杂，而是让你尽快完成以下闭环：

- 看懂价格
- 获取余额
- 创建 Key
- 发出第一条请求

只要这四步通了，后续接入自己的项目通常就会顺很多。
