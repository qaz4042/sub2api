# 运营前待确认配置示例

更新日期：2026-06-21  
适用项目：Sub2API  
当前实例：`portal.lizubin.online`  
参考文档：[运营前配置与数据检查清单](./运营前配置与数据检查清单.md)

## 一、使用说明

本文提供一套适合“个人运营、中国香港入口、3–10 人封闭测试”的起步示例。

- `[ ]` 表示尚未确认，修改完成后可改为 `[x]`。
- 标有“待填写”或“待确认”的内容必须根据实际业务修改。
- 模型成本和销售价格仅用于演示配置结构，不能直接用于正式计费。
- 第一阶段保持封闭注册、不接在线支付、不提供公开收费服务。

## 二、封闭试运行前需要确认

### 1. 公网入口

- [ ] 已确认正式公网入口方案。

```yaml
正式入口: 中国内地服务器 (暂时性) (已调通)
域名: portal.lizubin.online
访问协议: HTTPS only
HTTP处理: 301跳转HTTPS
#中国内地服务器用途: 不作为公网入口
```

### 2. 上游 URL 白名单

- [ ] 已填写全部实际使用的上游域名。
- [ ] 已确认不需要访问内网地址。
- [ ] 已确认代理失败后不允许回退直连。

```env
SECURITY_URL_ALLOWLIST_ENABLED=true
SECURITY_URL_ALLOWLIST_ALLOW_INSECURE_HTTP=false
SECURITY_URL_ALLOWLIST_ALLOW_PRIVATE_HOSTS=false
```

```yaml
允许的上游域名:
  - api.openai.com
  - api.anthropic.com

允许内网地址: 否
代理失败后直连: 否
```

只保留实际使用的平台域名，不要直接照搬示例域名。

### 3. 首批开放模型

- [ ] 已确定用户可见模型名称。
- [ ] 已确认每个模型的上游模型和计费模型。
- [ ] 已关闭图片、视频、音频及其他未核价模型。

```yaml
模型:
  - 用户模型名: Codex
    上游模型: Codex
    计费模型: 标准 1:1
    类型: 文本
    状态: 开启

暂不开放:
  - 图片模型
  - 视频模型
  - 音频模型
  - 未核价模型
  - 自动模型回退
```

### 4. 模型成本和销售价格

- [ ] 已使用真实上游账单核对成本。
- [ ] 已将汇率、支付手续费、代理和失败请求成本计入。
- [ ] 已确认销售倍率和目标毛利。
- [ ] 已确定账单误差阈值。

```yaml
text-standard:
  上游输入成本: 1.00元/百万Token
  上游输出成本: 4.00元/百万Token
  销售倍率: 1.30
  用户输入价格: 1.30元/百万Token
  用户输出价格: 5.20元/百万Token
  目标毛利率: 约23%

text-premium:
  上游输入成本: 10.00元/百万Token
  上游输出成本: 40.00元/百万Token
  销售倍率: 1.25
  用户输入价格: 12.50元/百万Token
  用户输出价格: 50.00元/百万Token
  目标毛利率: 约20%

账单允许误差:
  相对误差: 小于1%
  单笔绝对误差: 小于0.01元
```

### 5. 首批用户额度

- [ ] 已确定测试用户数量。
- [ ] 已确定每人余额、日额度和月额度。
- [ ] 已确认禁止透支。

```yaml
新用户默认余额: 0元
测试用户数量: 3人
每人测试余额: 20元
每日消费上限: 5元
单月最高额度: 20元
允许透支: 否
余额用完后: 停止调用
```

### 6. 限流参数

- [ ] 已确定用户、API Key、分组和上游账号四层限制。
- [ ] 所有 RPM 均为明确的非零值。
- [ ] API Key 必须绑定明确分组。

```yaml
用户:
  最大并发: 2
  RPM: 20
  每日额度: 5元

API_Key:
  最大并发: 2
  RPM: 15
  每日额度: 5元
  必须绑定分组: 是

分组:
  RPM: 40
  允许模型:
    - text-standard
    - text-premium

上游账号:
  最大并发: 3
  RPM: 30
  每日成本上限: 20元
```

### 7. 数据保留周期

- [ ] 已确定每类数据的保留周期。
- [ ] 已确认到期清理不会破坏计费和审计数据。

```yaml
原始使用记录: 30天
聚合统计: 365天
错误日志: 14天
登录及安全审计日志: 90天
备份操作记录: 365天
内容审核记录: 30天
支付订单: 暂不适用
```

### 8. 数据内容和用途

- [ ] 已确认默认不保存完整提示词和响应正文。
- [ ] 已确认凭证不会写入日志。
- [ ] 已明确计费、排障和审计数据范围。

```yaml
保存:
  - 用户ID
  - API Key ID，不保存完整Key
  - 请求模型
  - Token数量
  - 请求时间
  - 响应状态
  - 费用
  - 延迟
  - 上游账号ID

不保存:
  - 完整提示词
  - 完整响应正文
  - API Key
  - OAuth Token
  - Cookie
  - 代理密码

用途:
  计费: Token、模型、倍率、费用
  排障: 状态码、延迟、错误类型、请求ID
  审计: 登录、充值、额度修改、管理员操作
```

### 9. 告警设置

- [ ] 已确定主要和紧急告警渠道。
- [ ] 已确定告警接收人。
- [ ] 已测试容器停止、备份失败和磁盘告警。

```yaml
主要渠道: 管理员邮箱
紧急渠道: 企业微信机器人或Telegram
接收人: 你本人
静默时间: 无

立即告警:
  - 容器停止
  - HTTP健康检查连续失败3次
  - 数据库不可连接
  - 备份失败
  - 磁盘使用率达到90%

普通告警:
  - 磁盘使用率达到70%或80%
  - 内存持续超过85%
  - Swap持续增长
  - 上游成功率低于90%
  - TLS证书剩余有效期不足14天
```

### 10. 首批测试用户

- [ ] 已确定具体测试人员。
- [ ] 已告知测试人员不得转借账号和 API Key。
- [ ] 已安排至少 7 天测试周期。

```yaml
用户数量: 3人
用户来源:
  - 管理员本人
  - 1名熟悉API的测试者
  - 1名普通使用者

每人余额: 20元
测试周期: 7天
最低调用量: 每人20笔
开户方式: 管理员手动创建
禁止转借账号和API_Key: 是
```

每个上游账号复制并填写一份：

- [ ] 已确认账号来源及商业使用授权。
- [ ] 已确认是否允许额度分发或转售。
- [ ] 已准备停用条件和替换方案。

```yaml
账号名称: upstream-01

账号类型: 官方API或企业账号

负责人: 你本人
商业使用授权: 待确认
允许转售: 待确认
成本: 按量计费
当前额度: 100元
到期时间: 无
代理出口: 中国香港固定IP
可用模型:
  - text-standard
最大并发: 3
风险等级: 待填写，低/中/高
最近测试时间: 2026-06-21
停用条件:
  - 授权范围不明确
  - 连续失败率超过30%
  - 账号出现安全告警

```

## 三、公开运营前需要确认

以下内容不阻塞 3–10 人封闭试运行，但必须在开放注册或公开收费前完成。

### 1. 品牌和公开信息

- [ ] 已确定品牌名称、Logo 和副标题。
- [ ] 已发布文档、客服入口和服务状态页。

```yaml
网站名称: 示例 API
副标题: 稳定、透明的统一模型 API 服务
API_Base_URL: https://portal.lizubin.online/v1
文档地址: https://docs.example.com
客服邮箱: support@example.com
状态页: https://status.example.com
Logo: 待设计
```

### 2. 运营范围

- [ ] 已明确运营主体和实际业务开展地。
- [ ] 已明确服务器、用户及数据所在地区。
- [ ] 已确认跨境数据处理方式。

```yaml
运营主体: 个人，公开收费前调整为合适主体
服务器所在地: 中国香港
目标用户: 中国大陆及港澳地区成年用户
服务范围: API技术服务
数据存储位置: 中国香港
跨境数据传输: 存在，用户请求会发送至对应模型上游
未成年人服务: 不提供
```

### 3. 上游授权标准

- [ ] 所有用于公开服务的账号均有明确授权记录。

```yaml
允许使用:
  - 官方明确支持商业调用的API账号
  - 明确允许当前服务模式的企业账号

禁止使用:
  - 个人订阅共享
  - 来源不明账号
  - 禁止转售的额度
  - 绕过地区或平台限制的账号
```

### 4. 服务规则

- [ ] 已发布服务条款、隐私政策和可接受使用政策。
- [ ] 已明确违规分级及处理方式。

```yaml
禁止用途:
  - 违法内容
  - 欺诈和恶意自动化
  - 未授权网络攻击
  - API Key转售或公开共享
  - 绕过模型安全限制

违规处理:
  首次轻微违规: 警告
  严重违规: 立即暂停
  多次违规: 封禁账号
```

### 5. 退款规则

- [ ] 已由具备资质的专业人员结合实际运营模式审查。
- [ ] 已明确赠送、已消费及未消费余额的处理规则。

```yaml
人工赠送余额: 不退款
已消费余额: 不退款
未消费充值余额: 符合条件时人工审核退款
退款手续费: 从退款金额中扣除实际手续费
上游故障: 按实际错误账单退回余额
封禁账号: 合法未消费余额按条款处理
退款处理时间: 5至10个工作日
```

### 6. 注册方式

- [ ] 已确定从管理员开户切换到邀请或公开注册的条件。
- [ ] 开放注册前已配置邮件验证、密码找回和防刷。

```yaml
第一阶段: 管理员手动开户
第二阶段: 邀请码注册
公开注册: 暂不开启
邮箱验证: 开放邀请注册时启用
密码找回: 与邮箱验证同时启用
```

### 7. 商业模式

- [ ] 已确定计费模式、充值方式和最低充值金额。
- [ ] 尚未形成闭环的营销功能保持关闭。

```yaml
初期计费方式: 预付余额、按量扣费
订阅套餐: 暂不开启
在线支付: 暂不开启
优惠码: 关闭
邀请返利: 关闭
最低充值金额: 10元
余额透支: 禁止
```

### 8. 服务承诺

- [ ] 已确定客服响应时间和故障通知方式。
- [ ] 在有能力持续测量和履约前不承诺正式 SLA。

```yaml
正式SLA: 暂不提供
内部可用性目标: 99%
客服响应时间: 2个工作日内
故障通知: 状态页和邮件
计划维护通知: 尽量提前24小时
故障补偿: 根据影响范围人工判断
上游不可用责任: 不保证第三方上游持续可用
```

## 四、优先修改顺序

建议先确定以下八项，再开始封闭试运行：

1. 公网入口。
2. 实际上游平台和 URL 白名单。
3. 首批开放模型及模型映射。
4. 真实上游成本和销售倍率。
5. 测试用户余额、日额度和限流参数。
6. 告警渠道和接收人。
7. 数据保留周期。
8. 上游账号商业使用及转售授权范围。
