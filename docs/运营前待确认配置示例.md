## 一、使用说明

本文提供一套适合“个人运营、3–10 人封闭测试”的起步示例。

- `[ ]` 表示尚未确认，修改完成后可改为 `[x]`。
- 标有“待填写”或“待确认”的内容必须根据实际业务修改。
- 系统余额、使用记录、周报和月报继续使用原生 `USD` 单位。
- 对用户收款时固定按数字 `1 USD 系统余额 = 1 CNY 支付价格`，不进行汇率换算。
- 分组倍率固定为 `1.0`，不使用折扣、销售倍率或其他价格调整。
- 第一阶段保持封闭注册、不接在线支付、不提供公开收费服务。

#### 7.2 固定 1:1 计费规则

不使用折扣或额外销售倍率：

```yaml
分组名称: 标准计费
分组倍率: 1.0
其他用户专属倍率: 不设置
账号计费倍率: 1.0
图片独立倍率: 不启用
```

计算方式：

```text
模型标准费用（total_cost）× 1.0
  = 用户实际扣费（actual_cost）
  = 周报、月报显示费用
  = 相同数字的人民币价格
```

示例：

```text
标准费用 $10.00 × 1.0 = actual_cost $10.00
用户余额扣除 $10.00
运营口径下对应人民币价格 ¥10.00
```

#### 7.3 ChatGPT Plus 成本参考

```yaml
ChatGPT_Plus官方订阅价: 20 USD/月
是否包含API用量: 否
是否有官方每Token成本: 否
用途: 仅作为官方个人订阅价格参考
```

官方说明：[What is ChatGPT Plus?](https://help.openai.com/en/articles/6950777-what-is-chatgpt-plus)

ChatGPT Plus 与 OpenAI API 是两套独立产品。Plus 的每月订阅费不能换算成上述三个模型的官方 Token 成本，也不能作为 API 账单成本。个人 ChatGPT Plus 订阅不得作为未经授权的多用户 API 转售来源。

如果只把 Plus 的 `$20/月` 当作金额参照，则相同的 `$20` API 预算在“全部为非缓存输入”或“全部为输出”的极端情况下，大约对应：

| 模型 | $20 全部用于输入 | $20 全部用于输出 |
|---|---:|---:|
| `gpt-5.5` | 4.00M tokens | 0.67M tokens |
| `gpt-5.4` | 8.00M tokens | 1.33M tokens |
| `gpt-5.4-mini` | 26.67M tokens | 4.44M tokens |

这张表只是 API 价格换算，不代表 ChatGPT Plus 的消息额度、Token 配额或成本结构。

#### 7.4 成本风险

固定 `1 USD 系统余额 = 1 CNY` 只是你的销售规则，不会改变上游按真实美元收取的 API 费用。如果直接使用 OpenAI 官方 API，上游账单仍以美元结算，因此必须另行核对实际人民币成本和毛利。

例如，`gpt-5.5` 输出 100 万 Token 的官方 API 标准费用为 `$30`；按当前数字 1:1 规则，系统扣除 `30 USD` 余额，对应用户支付 `¥30`。这通常低于你实际支付的人民币成本，不能视为保本定价。

```yaml
账单允许误差:
  相对误差: 小于1%
  单笔绝对误差: 小于0.01 USD系统余额
```
