# 新手转化文案

更新日期：2026-06-27

适用位置：登录后 Dashboard、新手任务卡片、注册成功页、API Key 页面。

## 一、Dashboard 新手卡片

标题：

> 跑通你的第一条 API 请求

正文：

> 按下面四步完成接入。建议先用简单文本请求测试，确认链路可用后再接入真实项目。

步骤：

1. 充值或兑换余额。
2. 创建 API Key。
3. 复制 Base URL。
4. 运行 curl 示例。

辅助链接：

- 查看费用公告
- 查看使用指南
- 常见问题
- 联系客服

## 二、API Key 页面提示

标题：

> 每个项目建议使用独立 API Key

正文：

> 独立 Key 更方便统计用量、排查问题和处理泄露风险。不要把 Key 写进前端代码，也不要提交到公开 Git 仓库。

操作提示：

- 新项目创建新 Key。
- 临时测试使用单独 Key。
- 怀疑泄露时立即删除旧 Key。
- 团队成员离开后及时轮换相关 Key。

## 三、Base URL 说明

标题：

> 只需要替换 Base URL

正文：

> 如果你的工具或 SDK 支持 OpenAI 风格接口，通常只需要把默认 OpenAI Base URL 替换为本站提供的地址，再填入你的 API Key。

示例说明：

```text
Base URL: https://your-domain.com/v1
API Key: 在站内 API Key 页面创建
```

## 四、余额页提示

标题：

> 充值金额、站内余额和模型扣费是三个概念

正文：

> 充值金额是支付口径，站内余额是账户结算口径，模型扣费按实际调用消耗计算。正式使用前建议先阅读费用公告。

辅助链接：

- 费用公告
- 使用记录
- 订单记录

## 五、首次调用成功提示

标题：

> 接入成功

正文：

> 当前账户、API Key、Base URL 和模型配置已通过基础验证。接入真实项目时，建议先控制请求规模，并持续观察使用记录和余额变化。

下一步：

- 接入你的应用。
- 为不同项目创建独立 Key。
- 查看模型费用和用量记录。

## 六、首次调用失败提示

标题：

> 请求未成功，先检查这几项

检查项：

- API Key 是否复制完整。
- Base URL 是否填写正确。
- 模型名是否在站内开放。
- 账户余额是否充足。
- 请求格式是否兼容 OpenAI 风格接口。

提示：

> 建议先复制使用指南中的 curl 示例做最小测试，再排查业务代码。
