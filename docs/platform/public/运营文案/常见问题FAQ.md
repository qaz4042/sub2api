# 常见问题 FAQ

更新日期：2026-06-27

适用位置：帮助中心、客服自助回复、用户群公告。

## 一、本站提供的是什么服务？

本站提供 OpenAI API 官方直连接入能力，以及 API Key 管理、余额扣费、用量记录和基础接入说明。

你可以把它理解为一个更适合开发者和小团队使用的 OpenAI 风格 API 网关。

## 二、本站是官方 OpenAI 网站吗？

不是。本站不是 OpenAI 官方网站，也不代表 OpenAI。

本站提供的是基于 OpenAI 官方 API 的接入、计费、管理和辅助服务。模型能力、价格口径和可用性会受到 OpenAI 官方 API 的影响。

## 三、为什么要使用本站？

适合以下用户：

- 希望快速获得 OpenAI 风格 API 接入能力。
- 不想自己处理账号、网络、支付和基础配置细节。
- 需要为不同项目创建不同 API Key。
- 希望查看用量记录和余额变化。
- 小团队希望统一管理 AI API 成本。

## 四、是否需要自己准备 OpenAI 账号？

使用本站公开接入服务时，通常不需要你自己准备 OpenAI 账号。

如果你选择私有部署或团队方案，也可以按需求采用 BYOK 模式，也就是使用你自己或团队已经合规持有的 API Key。

## 五、是否支持 Codex App 和 CC Switch？

支持使用支持自定义 Base URL 的工具接入。Codex App 配合 CC Switch 的接入流程，可查看开发者解决方案文档。

建议首次接入时先创建独立 API Key，并用简单任务确认链路可用。

## 六、充值金额和模型扣费为什么不一样？

充值金额是支付口径，站内余额是账户结算口径，模型扣费按实际 Token 消耗计算。

例如一次请求的费用会受到模型、输入长度、输出长度、缓存命中和站内倍率等因素影响。具体以费用公告和站内使用记录为准。

## 七、为什么一次调用比预期更贵？

常见原因包括：

- 输入上下文过长。
- 输出内容过多。
- 使用了高单价模型。
- 把充值价格和模型 Token 价格混在一起理解。
- 请求中包含大量历史对话或文件内容。

建议先用轻量模型和短输入测试，再接入真实项目。

## 八、请求失败会扣费吗？

一般原则是：未形成有效消耗的请求不应按成功请求扣费。

但实际处理需要结合请求是否已进入上游处理、是否产生 Token 消耗、是否出现流式中断等情况核查。具体以售后与异常处理规则为准。

## 九、Key 泄露了怎么办？

请立即执行以下操作：

1. 删除或禁用泄露的 API Key。
2. 创建新的 API Key。
3. 检查近期使用记录。
4. 从代码仓库、日志、截图和配置文件中移除旧 Key。

因用户自身泄露导致的消耗，通常不属于站内赔付范围。

## 十、是否承诺所有模型长期可用？

不承诺。

模型开放范围会受到官方模型上下线、价格变化、接口能力、稳定性和运营成本影响。实际可用模型以站内模型列表和公告为准。

## 十一、是否支持私有部署？

支持。

私有部署适合希望在自有服务器或云账号中部署 AI API 网关的团队。可根据实际需求配置 API Key、分组、额度、模型、日志、备份和管理后台。

## 十二、如何联系客服？

请优先通过站内公开客服入口或公告中列出的联系方式咨询。

咨询时建议提供：

- 账户标识。
- 请求时间。
- API Key 名称或备注。
- 请求模型。
- 错误信息或 request_id。

不要在公开群聊中发送完整 API Key。
