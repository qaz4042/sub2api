# 运营文案总览

更新日期：2026-06-27

本目录用于沉淀 Sub2API 对外运营可复用文案，适合放到官网、帮助中心、公告、客服回复、渠道推广和私有部署介绍中。

## 一、统一定位

推荐长期使用的核心定位：

> 面向开发者和小团队的 OpenAI API 网关，提供官方直连接入、透明计费、API Key 管理、用量统计和私有部署支持。

对外表达建议围绕四个关键词：

- 官方直连
- 计费透明
- 解决方案

## 二、不建议使用的话术

以下话术短期可能带来流量，但容易增加误解、投诉和合规风险，不建议作为公开主卖点：

- 官方几折
- 无限量
- 永不封号
- 零风控
- 全网最低价
- 订阅账号共享池
- 绕过官方限制

## 四、用户分层

### 个人开发者

关心点：

- 能不能快速接入。
- 价格和余额怎么理解。
- Codex App / OpenAI SDK / 常见工具能不能用。
- 不想处理账号、网络、支付和配置细节。

推荐表达：

> 适合希望快速拿到 OpenAI 风格 API 接入能力的开发者。注册后创建 API Key，复制 Base URL，即可用常见 OpenAI SDK 或支持自定义 Base URL 的工具调用。

### 小团队

关心点：

- 多人共用 Key 是否可控。
- 用量能不能统计。
- 成本能不能追踪。
- 密钥泄露后能不能快速止损。

推荐表达：

> 适合 3-30 人团队统一管理 AI API 接入。可以按项目创建 API Key，查看用量记录，控制额度和调用范围，减少多人各自管理 Key 带来的混乱。

### 企业或工作室

关心点：

- 是否支持私有部署。
- 是否能接入现有系统。
- 是否能控制数据、权限和审计。
- 是否有交付、运维和文档。

推荐表达：

> 支持部署到客户自有服务器或云账号中，按团队实际业务配置模型、分组、额度、API Key、日志和备份策略，交付可验收的 AI API 网关。

## 五、当前阶段建议

当前阶段优先把“看懂、接入、查账、出问题处理”四件事做好：

1. 首页说清楚服务定位。
2. 注册后引导用户跑通第一条请求。
3. 费用公告解释充值、余额和模型扣费。
4. FAQ 覆盖高频问题。
5. 售后规则公开异常处理边界。

增长不应先靠复杂活动，而应先靠清楚文档和稳定体验降低用户决策成本。
