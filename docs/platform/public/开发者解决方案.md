# 开发者解决方案

更新日期：2026-06-24
适用对象：Windows 开发者，使用 Codex App 连接本站 API 进行开发

## 一、准备条件

- 推荐使用 Windows 11 或以上版本。
- 准备一个可登录本站的账号。
- 确认本机网络可以正常访问 Codex App、CC Switch 和本站门户。

相关入口：

- Codex App：<https://apps.microsoft.com/detail/9nt1r1c2hh7j?hl=zh-CN&gl=CN>
- CC Switch 官网：<https://ccswitch.io/zh/>
- CC Switch GitHub Releases：<https://github.com/farion1231/cc-switch/releases>
- 本站门户：<https://portal.lizubin.online/>

## 二、安装 Codex App

1. 打开 Microsoft Store 中的 Codex App 页面。
2. 点击获取或安装，按提示完成安装。
3. 安装完成后先启动一次，确认应用可以正常打开。

如果 Microsoft Store 打不开，可以先检查 Windows 版本、系统区域、网络代理和 Microsoft Store 登录状态。

## 三、安装 CC Switch

1. 打开 CC Switch 官网或 GitHub Releases 页面。
2. 下载 Windows 安装包。
3. 运行安装包并按提示完成安装。
4. 安装完成后启动 CC Switch，确认托盘或主窗口可以正常显示。

如果已经拿到 `CC-Switch-v3.16.3-Windows.msi`，可直接运行该 MSI 安装包。

## 四、注册登录并导入 API

1. 打开本站门户：<https://portal.lizubin.online/>
2. 注册并登录账号。
3. 进入 API Key 管理页面，创建或选择一个可用的 API 密钥。
4. 点击页面中的“导入到 CC Switch”。
5. 系统会将当前 API 配置导入到 CC Switch。

导入后建议打开 CC Switch 确认新增配置已经出现。

## 五、切换 API 并重启 Codex

1. 打开 CC Switch。
2. 切换到刚刚导入的 API 配置。
3. 完全退出 Codex App。
4. 重新启动 Codex App。
5. 在 Codex App 中发起一次简单开发任务，确认链路可用。

完成以上步骤后，即可使用 Codex App 连接本站 API 进行开发。

## 六、macOS 安装与使用

macOS 用户也可以使用同样的接入流程：

1. 下载并安装 macOS 版本的 Codex App。
2. 从 CC Switch 官网或 GitHub Releases 下载 macOS 版本。
3. 安装并启动 CC Switch。
4. 打开本站门户：<https://portal.lizubin.online/>
5. 注册登录后创建或选择 API 密钥。
6. 点击“导入到 CC Switch”，将配置导入 CC Switch。
7. 在 CC Switch 中切换到新导入的 API。
8. 完全退出并重新启动 Codex App。
9. 在 Codex App 中发起一个简单开发任务，确认链路可用。

macOS 注意事项：

- 如果系统提示应用来自未识别开发者，可在“系统设置 -> 隐私与安全性”中允许打开。
- 如果 Codex App 没有走到 CC Switch 配置，先确认 CC Switch 已启动，并检查系统代理或网络权限。
- 如使用 Clash、Surge、Mihomo 等代理工具，建议确认 Codex App 和 CC Switch 的流量都走同一条可用链路。

## 七、排查建议

如果 Codex App 无法正常调用，优先检查：

- CC Switch 当前是否选中了新导入的 API。
- API Key 是否仍然有效，账户是否有可用额度。
- Codex App 是否已经重启。
- Windows 代理、TUN 或安全软件是否拦截了请求。
- 本站门户是否可以正常登录和访问。

建议先用一个最小任务验证，例如让 Codex App 修改一个简单文本文件，再逐步接入真实项目。
